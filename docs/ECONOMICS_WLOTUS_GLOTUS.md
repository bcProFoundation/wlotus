## Thesis

| Token | Ticker | Role | Monetary? | Mint posture |
|-------|--------|------|-----------|--------------|
| **wLotus** | `WLOTUS` (test: `dWLOTUS`) | Memorial / dana proof | **No** (ceremonial) | Temple-centric split; mobile may get **sponsored XEC fees** |
| **Golden Lotus** | `GLOTUS` | Scarce burnable value for special events & later commerce | **Yes** (real cost) | **Permissionless** remint; miner pays own XEC; **no platform mint tax** |

People **offer for wLotus**. wLotus is generated by PoW remint. **GLOTUS** is the economic layer (premine + open mining + event burns).

```
Offer (mobile, temple-sponsored fees)
        │
        ▼
   Mint 108 wLotus (one mala)
        ├── 1   → miner / offerer
        └── 107 → temple (wlotus.org)  — fee reimbursement & desk inventory

Burn / give wLotus          ← scalable path for non-miners (transfer then offer)
Mine GLOTUS                 ← open, miner pays XEC; burn on special events
```

## wLotus

### Issuance split (covenant-enforced)

Each successful remint mints **108** atoms — **one mala round** (108 beads):

| Share | Destination | Purpose |
|------:|-------------|---------|
| **1** | Miner / offerer | Presence bead — kept or later burned/given |
| **107** | Temple **P2SH** (IFP-style multisig / cold) | Rest of the mala — fee reimbursement & inventory |

Anyone may satisfy the covenant (including copied clients) **if they pay their own XEC fees** and accept the **107→temple / 1→miner** outputs (temple keeps **≈99%** of each mala).

**Launch custody:** temple sink is **P2SH**. Redeem is revealed only on spend. See [Temple custody](#temple-custody).

### Why 107 / 108 (mala)

- **108** = one full mala; **1** presence bead + **107** dana for the sangha / desk.
- Sponsored **mobile** offerings pay XEC from the desk; the 107 replenish that desk.
- Independent miners who pay their own fees still forfeit 107/108 — so **sponsored mobile remains advantaged** without claiming exclusive software.
- wLotus is **not** money; temple concentration is acceptable for a ritual ledger.

### Product intent (anti-farm + presence)

| Mechanism | Role |
|-----------|------|
| **1 miner / 107 temple** | **Practical anti-farming.** Independent remints pay XEC and keep only **1/108**. Temple-sponsored Offer pays **0 XEC** for the user and still fills the desk with **107**. |
| **Soft pray timer** (`VITE_MIN_PRAY_SECONDS`, default 60) | **Attention tax after remint.** PoW → remint ASAP (tip race) → hold → memorial burn. Cancel skips burn; desk keeps the miner atom. |
| **PoW bits (base 0, whole-byte)** | Launch presence is **soft pray + 1/107 + fees**, not hashrate. Base **0** maximizes headroom to the 128 sunset. Whole-byte only (`bits % 8 == 0`). |
| **Moore clock (+1 bit / 500 days)** | 五百罗汉 — intentional dearening for **WLOTUS** as bootstrap ritual stock. See [CLOCK.md](./CLOCK.md). |
| **128-bit hard sunset** | Covenant `verify bits <= 128` — when calendar would require bits **> 128**, remints **fail**. WLOTUS becomes legacy; **GLOTUS** carries the living economy. |
| **Desk tips at launch** | Serve **1** tip first. Genesis still **28** batons so the desk can raise `MINT_SERVING_TIP_COUNT` if needed. |
| **Pagoda burn → remint loop** | Memorial burns recirculate demand while Moore makes fresh mining dearer. Desk inventory (107/108) bridges non-miners. |
| **Token network hashrate** | Does **not** secure WLOTUS (ownership rides **eCash**). |

**Long-term thesis:** start at **base 0** so the 128-bit sunset lasts ~**175 years**; scarcity ticks on a 500-day arhat calendar — **WLOTUS bootstraps**, then **dries toward GLOTUS**. Soft pray + baton caps + 1/107 carry early presence/anti-farm.

**Do not claim:** “more WLOTUS hashrate = safer token.”  
**Do claim:** sponsored Offer wins vs commercial fee-paying miners on **XEC cost**; sincere users pay **attention**; non-miners **buy / receive / burn**.

### Does 1/107 deter fee-paying miners? (yes, on XEC cost)

Reference fee: **~5.46 XEC** per remint. Burn-after-mint adds a second fee → **~11 XEC** desk cost per sponsored remint that leaves **107** temple inventory atoms.

| Actor | XEC out | wLotus kept | Implied cost / lotus |
|-------|--------:|------------:|---------------------:|
| **Temple desk** (sponsor remint + burn) | ~11 | **107** inventory | **~0.10 XEC** (`11 / 107`) |
| **Independent miner** (pays own remint; keeps the 1) | ≥ ~5.46 | **1** | **≥ ~5.46 XEC** |

1. **Farming for inventory is uneconomic** vs buying from the desk below ~5.46 XEC / WLOTUS.
2. Desk cost basis **~0.10 XEC / lotus**. Selling at **1 XEC / WLOTUS** is still profitable vs cost and far below the independent miner’s break-even.
3. Soft timer is **client / mint-api policy**, not a covenant rule — permissionless miners skip it and still lose on **1/107 + fees**.

### Soft timer (attention), not hashrate security

Phone electricity ≪ XEC fee. Ritual value is **minutes of presence**, not SHA256d network work.

- Official Offer: device PoW → **remint immediately** → soft pray floor → **memorial burn**.
- Cancel during the soft wait **skips the burn**; the miner atom stays with the desk.
- Soft timer must **not** delay remint — that would lose tip races.

### Non-miners and scale

1. Temple (or others) **transfer** wLotus to someone who will not mine.
2. Recipient **offers / burns** on-platform.

Burning/giving is the scalable ritual; mining is issuance + presence tax.

### Capped free-mine issuance vs pagoda burn demand

Batons cap **how fast** new atoms appear; pagoda burns are the **demand** valve. Pagoda keeps **107/108 ≈ 99%** of each remint.

| Condition | Formula | Meaning |
|-----------|---------|---------|
| **XEC cashflow ≥ 0** | \(B \ge 11 R / P\) | Sale revenue covers sponsor fees |
| **Desk inventory flat** | \(B \ge 107 R\) | Burns offset temple refill |
| **Price parity (mine floor)** | desk ask \(P \to 5.46\) | Buyers pay what an independent miner burns in fees for **1** atom |

\(R\) = remints/day (≤ baton ceiling). \(B\) = atoms burned/day from purchased desk stock.

| Setup | \(R_{\max}\)/day (\(T{\approx}10\)s) | Temple +107/day | Burns/day to flat desk |
|-------|-------------------------------------:|----------------:|-----------------------:|
| Desk **1** tip | ~**8.6k** | ~**920k** | ~**920k** |
| Full **28** tips | ~**242k** | ~**26M** | ~**26M** |

**Practical scarcity** comes from remints **not** sitting at the ceiling while memorial burns continue. Moore is optional reinforcement; demand > issuance can appreciate even if PoW stays easy.

### Mobile fee policy (off-chain)

| Who | XEC remint fee |
|-----|----------------|
| Official mobile / mint-api | Temple may **sponsor** (rate-limited) |
| Anyone else | Pays own XEC; still must mint **107→temple, 1→self** |

### Convertibility

wLotus ↔ GLOTUS **may** exist later at a **very large** fixed ratio. Prefer **defer** at launch (avoids a shadow price on ceremonial wLotus).

## Golden Lotus (`GLOTUS`)

### Role

- Higher economic weight than wLotus.
- Burned on **special events** (real sacrifice / ticket-like cost).
- Suitable later as a commerce unit **because mint is not temple-taxed**.

### Premine

An initial **premine** (disclosed %) funds development, marketing, and onboarding. Publish allocation + vesting.

### Mint policy

| Option | Verdict |
|--------|---------|
| Platform cut on every GLOTUS mint | **Reject** for launch |
| Permissionless mint; miner pays XEC only | **Adopt** |
| Temple revenue | Premine + event burns + optional *services* priced in GLOTUS |

### Openness

GLOTUS mining is **permissionless**. No requirement to pay wLotus or route mint outputs to the temple.

## Temple custody

| Layer | Address type | Role |
|-------|--------------|------|
| Temple (in covenant) | **P2SH** (multisig / cold) | Receives 107/remint |

## Implementation notes

- Covenant: `WlotusPowRemintMooreTipTemple` — mint **108**; **1** miner P2PKH + **107** temple P2SH.
- **Memorial:** burn the miner **1** after remint (`DANA` LOKAD on burn tx).
- Genesis: `baseZeroBits = 0`, Moore **500 d**/bit. Only `TICKER` differs (`dWLOTUS` vs `WLOTUS`).
- Mint-api / web: temple remint → burn miner atom; `memorialOnBurn: true`. Soft tip count `MINT_SERVING_TIP_COUNT=1` at launch.
- Soft pray: `VITE_MIN_PRAY_SECONDS` on the Offer client.
- GLOTUS: separate token; open remint; no temple mint tax.

Settled intent: anti-farm = **1/107 + fees**; presence = **soft timer + base 0 Moore ramp**.
