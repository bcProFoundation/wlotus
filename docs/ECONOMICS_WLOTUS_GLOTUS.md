# Economics — WLotus (temple) + Golden Lotus (open)

**Status:** design decision (2026-07-20). Launch path favors **WLotus first**, not Prayer-as-money.  
Related: [VISION.md](./VISION.md), [ECONOMICS.md](./ECONOMICS.md), [ECONOMICS_PRAYER.md](./ECONOMICS_PRAYER.md) (phone bootstrap experiments).

---

## Thesis

| Token | Role | Monetary? | Mint posture |
|-------|------|-----------|--------------|
| **WLotus** | Prayer / memorial / dana proof on the temple | **No** (ceremonial) | Temple-centric split; mobile may get **sponsored XEC fees** |
| **Golden Lotus** (`GLotus` / “Golden Lotus”) | Scarce burnable value for special events & later commerce | **Yes** (real cost) | **Permissionless** remint; miner pays own XEC fees; **no platform mint tax** |

People **pray for WLotus**. WLotus is generated by PoW remint. **Golden Lotus** is the economic layer (premine + open mining + event burns).

```
Pray / Offer (mobile, temple-sponsored fees)
        │
        ▼
   Mint 100 WLotus
        ├── 1  → prayer (miner / offerer)
        └── 99 → temple (wlotus.org)  — fee reimbursement & desk inventory

Burn / give WLotus          ← scalable path for non-miners (transfer then offer)
Mine Golden Lotus           ← open, miner pays XEC; burn on special events
```

## WLotus

### Issuance split (covenant-enforced output shape)

Each successful remint mints **100** WLotus atoms:

| Share | Destination | Purpose |
|------:|-------------|---------|
| **1** | Prayer / miner | Presence reward — kept or later burned/given |
| **99** | Temple **P2SH** (IFP-style multisig / cold) | Fee reimbursement & inventory; spends are rare ops, not daily key hops |

This is **not** “app-only mining.” Anyone may satisfy the covenant (including copied clients) **if they pay their own XEC fees** and accept the **99→temple / 1→miner** outputs. The temple cannot stop that on a public chain — and does not need to.

**Launch custody:** temple sink in Script is **P2SH** (dryrun and production), same pattern as eCash’s baked IFP address. Redeem is revealed only on spend; quantum threats are handled by a future address-format fork, not by baking a single hot P2PKH. See [Temple custody](#temple-custody-dryrun--launch-decision).

### Why 99% temple

- Sponsored **mobile** offerings pay XEC from the temple desk; the 99 WLotus replenishes that desk (and funds inventory).
- Independent miners who pay their own fees still forfeit 99/100 — so **sponsored mobile remains advantaged** without claiming exclusive software.
- WLotus is **not** positioned as money; temple concentration is acceptable for a ritual ledger.

### Non-miners and scale

Mining does not scale to every memorial. Preferred path:

1. Temple (or others) **transfer** WLotus to someone who will not mine.
2. Recipient **offers / burns** on-platform (memorial + dana).

Burning/giving is the scalable ritual; mining is the issuance + presence tax.

### Mobile fee policy (off-chain)

| Who | XEC remint fee |
|-----|----------------|
| Official mobile / mint-api path | Temple may **sponsor** (rate-limited) |
| Anyone else | Pays own XEC; still must mint **99→temple, 1→self** |

Caps, installId limits, and PoW difficulty remain product controls on the free lane — not chain exclusivity.

### Convertibility

WLotus ↔ Golden Lotus **may** exist later at a **very large** fixed ratio (ritual bridge), but:

- Prefer **defer** convertibility at launch (avoids a shadow price on “non-monetary” WLotus).
- If added: mechanical on-chain burn/convert, disclosed ratio, rare — not an admin soft peg.

## Golden Lotus

### Role

- Higher economic weight than WLotus.
- Burned on **special events** (real sacrifice / ticket-like cost).
- Suitable later as a commerce unit **because mint is not temple-taxed**.

### Premine

An initial **premine** (disclosed %) funds development, marketing, and onboarding. Publish allocation + vesting; opacity is the reputation risk, not premine itself.

### Mint policy (decision)

| Option | Verdict |
|--------|---------|
| Platform cut on every GLotus mint | **Reject** for launch — centralizes the money-like coin and weakens commerce trust |
| Permissionless mint; miner pays XEC only | **Adopt** |
| Temple revenue | Premine + event burns + optional *services* priced in GLotus — not a mandatory mint tax |

Optional later: a separate “blessed / faster” **service lane** — never a hard covenant tax on all GLotus.

### Openness

Golden Lotus mining is **permissionless**. No requirement to pay WLotus or route mint outputs to the temple.

### Temple custody (dryrun + launch decision)

**Covenant destination: temple P2SH** (immutable `templeScriptHash` — 99 atoms per remint).

| Layer | Address type | Role |
|-------|--------------|------|
| Temple (in covenant) | **P2SH** (multisig / cold) | Receives 99/remint; IFP-style baked sink |
| Spends | Redeem + keys | Rare, intentional treasury moves — not a daily hot key |

**Why P2SH (not hot P2PKH):** ops custody matches other eCash baked funds (e.g. IFP). Multisig reduces single-key loss; the redeem is public only when spending. Quantum / address-format risk is accepted the same way as other P2SH sinks — migrate via a **protocol fork** to a new format when needed, rather than encoding a forever-hot P2PKH.

**Dryrun convenience:** if `TEMPLE_ADDRESS` is unset, `create-dryrun-token` P2SH-wraps the genesis wallet’s P2PKH so the same key can spend by revealing redeem (`templeRedeemHex` in the deployment JSON). Launch should set `TEMPLE_ADDRESS` to a real multisig P2SH.

```bash
# Launch / Contabo: bake a known multisig P2SH
TIER=wlotus TEMPLE_ADDRESS=ecash:p… BATONS=2 npm run create-dryrun-token
```

If temple keys are compromised: stop mint-api, move remaining inventory with the redeem, serve a **new** covenant with a new `templeScriptHash`.

## Implementation notes

- Covenant: `WlotusPowRemintMooreTipTemple` — **P2SH** temple (99) + miner P2PKH (1); mint = 100. Fits ≤520 B / ≤201 ops (no memorial EMPP on mint — op budget).
- **Memorial:** burn the miner **1** atom after remint (`WLBR` on burn tx). The on-chain burn is the gift — memorial and dana offered for all.
- Dryrun:
  ```bash
  TIER=wlotus BATONS=28 TEMPLE_ADDRESS=ecash:p… npm run create-dryrun-token
  TIER=wlotus BATON_INDEX=0 npm run mine-dryrun-once
  ```
- Mint-api / web: temple remint → burn miner atom; `memorialOnBurn: true`. Soft tip count `MINT_SERVING_TIP_COUNT=2`.
- Bits: whole-byte only; see [CLOCK.md](./CLOCK.md). Base **24** for WLotus dryrun.
- Golden Lotus: separate token; open remint; no temple mint tax.

Open product decisions: bits/difficulty, premine %, cold multisig policy, convertibility, memorial-on-mint if op budget frees up.
